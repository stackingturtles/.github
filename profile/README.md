# Stacking Turtles 🐢

**Agentic software development studio — Glasgow, UK.**

We build financial services, e-commerce, blockchain, and developer tooling products with a simple rule: agents can write the code, but acceptance criteria write the rules.

## 🙋‍♀️ What are we about?

Stacking Turtles is a boutique software studio focused on shipping useful, well-specified software. We combine senior engineering judgement with agentic development workflows: structured user stories, explicit acceptance criteria, automated verification, and tight feedback loops.

Our work spans:

- **Financial services and digital assets** — robust systems where correctness, auditability, and operational clarity matter.
- **Blockchain and DeFi tooling** — smart contract integrations, oracle infrastructure, and cross-chain/product architecture.
- **E-commerce and reseller tools** — practical software for pricing, listings, and operational speed.
- **Developer tooling and AI agents** — tools that help engineering teams turn vague intent into executable, testable work.

Some of the things we have built:

## 🌈 Contribution guidelines

Most of our active client work is private, but we open-source tools and supporting libraries when they are useful outside our own projects.

If you want to get involved:

1. **Start with the README** in the repository you care about.
2. **Open an issue before large changes** so we can agree the problem, acceptance criteria, and shape of the solution.
3. **Keep pull requests focused** — one clear change, with tests or verification notes.
4. **Write acceptance criteria** for behaviour changes. “Works better” is not enough; tell us what should be true when the work is done.
5. **Prefer boring reliability over cleverness** unless the cleverness buys something measurable.

Good contributions usually include:

- a short explanation of the problem;
- the chosen approach and trade-offs;
- tests, screenshots, logs, or reproduction steps where relevant;
- documentation updates when behaviour changes.
